# Contributing

Спасибо за интерес к ПриДел.

## Как участвовать

- сообщать о багах и регрессиях;
- предлагать улучшения интерфейса и бизнес-логики;
- присылать исправления с тестами;
- улучшать документацию и сценарии запуска в MAX.

## Локальная разработка

```bash
cp .env.example .env
make up
make migrate
make seed
make test
```

## Правила для изменений

- не ломать существующие сценарии смет, ролей, оплат и согласований;
- сопровождать нетривиальные изменения тестами;
- сохранять разделение на delivery, services, core и models;
- избегать смешивания каналовой логики с доменной логикой.

## Pull request checklist

- [ ] код проходит тесты;
- [ ] документация обновлена, если поведение изменилось;
- [ ] не добавлены секреты и токены;
- [ ] нет случайных изменений вне задачи.
